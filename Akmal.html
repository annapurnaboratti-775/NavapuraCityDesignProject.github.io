üéØ Algorithm-to-Application Mapping

Algorithm / DS Primary Use Why This Algorithm? Alternative
Dijkstra's Algorithm Ambulance routing with real-time traffic Guarantees shortest weighted path; handles dynamic edge weights A* (with heuristic), Bellman-Ford (negative weights)
BFS Evacuation route calculation Finds shortest unweighted paths; simple and fast for uniform distances Dijkstra (if weighted), DFS (non-optimal)
A* Search Fire truck navigation with obstacles Heuristic-guided; faster than Dijkstra when goal-directed Dijkstra, Greedy Best-First (non-optimal)
Hash Table Vehicle status lookup O(1) average-case lookup; critical for real-time dispatch Binary Search Tree (O(log n)), Array (O(n))
Priority Queue (Heap) Dispatching nearest vehicle Efficiently retrieves min/max element; supports dynamic updates Sorted Array (slower updates), BST
Segment Tree Disaster risk range queries O(log n) range queries; supports interval updates Fenwick Tree (prefix only), Array (O(n))
Fenwick Tree Crime frequency queries Space-efficient; simpler than Segment Tree for cumulative data Segment Tree (more features), Prefix Sum Array (static)
AVL Tree Fire incident history storage Self-balancing; guarantees O(log n) operations Red-Black Tree, B-Tree (database), Hash Table (no order)
