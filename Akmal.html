System Overview

SERCC integrates ambulance, fire, and police services into a unified platform, processing over 1,000 emergency calls daily with 150+ vehicles.

Implementation Steps (ADS)

  • Model junctions, hospitals, stations as nodes in a weighted graph.
  • Store vehicle status in hash tables for O(1) lookup.
  • Use priority queues to dispatch based on proximity and severity.
  • Run BFS/DFS to ensure regional reachability.

Algorithms / Data Structures Used

Graph (Adjacency List) Hash Table Priority Queue (Heap) BFS / DFS
